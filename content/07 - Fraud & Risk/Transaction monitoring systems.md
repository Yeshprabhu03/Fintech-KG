# Transaction monitoring systems

> ⚠️ Draft — needs verification against primary sources

## Summary
The automated oversight systems used by financial institutions to detect, flag, and prevent suspicious financial activity in real-time or near real-time.

## Key Concepts
- **Rules-Based Engine**: Setting specific thresholds (e.g., flag any transfer >$10k) to trigger a manual review.
- **Machine Learning Detection**: Using past fraud patterns to predict the probability of a current transaction being fraudulent.
- **False Positives**: The operational burden of legitimate users being blocked by over-sensitive monitoring systems.
- **Rule Management**: The ongoing process of updating detection logic to react to new fraud trends (e.g., the rise of "pig butchering" scams).
